## 实测关键发现

### 1. /cpbd/ 和 /y/ 路径在 8:00 前必返回 403

可能是编辑团队尚未发布当日内容（这些是每日 8:00 发布的固定栏目）。早盘快报触发时间若在 8:00 之前，需做好 403 的预案。

### 2. /roll/ 路径是稳定可靠的早盘消息源

- 单页可列出 30+ 篇 6/30 当日文章（标题+链接）
- 单篇文章可用 `<!-- 正文开始 -->` 标记提取 1000-3000 字符正文
- 覆盖：宏观政策、产业新闻、公司公告、环球市场、机器人出海、汽车新国标等

### 3. JS 渲染站点 200 OK 但无内容

金十数据、财联社、格隆汇等站点的 HTTP 状态码正常，但实际快讯/电报内容由前端 JavaScript 渲染。`urllib.request` 抓取仅返回导航/链接骨架。

**绕过策略**：用 Sina `/roll/` 替代，覆盖绝大多数快讯内容。

## 早盘快报推荐数据采集流程（更新版）

1. **互斥锁检查**：`~/.hermes/cron/locks/早盘_$(date +%Y-%m-%d).lock`
2. **读取昨日复盘 JSON**：`/usr/local/files/docs/stock/$(date -d 'yesterday' +%Y-%m-%d)-A股复盘.json`
3. **美股指数**：腾讯 API `qt.gtimg.cn/q=usDJI,usIXIC,usINX`
4. **A 股指数（昨日收盘）**：腾讯 API `qt.gtimg.cn/q=sh000001,sz399001,sz399006,sh000016,sh000688`
5. **消息面**：新浪 `/stock/` 首页 → 提取 6/30 文章链接 → 逐篇 GET 提取 `<!-- 正文开始 -->` 正文
6. ~~**Sina /cpbd/ 操盘必读**~~（403 跳过）
7. ~~**Sina /y/ 财经早报**~~（403 跳过）
8. ~~**金十数据 / 财联社 / 格隆汇**~~（JS 渲染，无内容）
