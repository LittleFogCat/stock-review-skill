# A股涨跌停阈值与涨停股统计

## 涨跌停阈值（A 股市场，按板块代码前缀区分）

A 股市场**不是所有股票 10% 涨跌停**，必须按代码前缀区分阈值，否则涨停/跌停统计会全部错误：

| 板块 | 代码前缀 | 涨停阈值 | 跌停阈值 |
|------|---------|---------|---------|
| 沪深主板 | 60xxxx、00xxxx（6 位） | ±10% | -10% |
| 创业板 | 300xxx、301xxx | ±20% | -20% |
| 科创板 | 688xxx | ±20% | -20% |
| 北交所 | 8xxxxx（83/87/88 等） | ±30% | -30% |
| ST/*ST 股 | 名称带 ST 前缀 | ±5% | -5% |

> **注意**：ST 股阈值是 ±5%，不是 10%。如果一个 *ST 股（如 *ST 仕净）20% 涨幅是异常事件（如重组），不是涨停。

## 精确统计涨停/跌停的 Python 函数

```python
def is_limit_up(code: str, pct: float, is_st: bool = False) -> bool:
    """判断是否涨停（含一字板）"""
    if pct is None:
        return False
    if is_st:
        return pct >= 4.95
    if code.startswith(("300", "301", "688")):
        return pct >= 19.5  # 留 0.5 容差
    if code.startswith("8") and len(code) == 6:
        return pct >= 29.5
    return pct >= 9.7

def is_limit_down(code: str, pct: float, is_st: bool = False) -> bool:
    """判断是否跌停"""
    if pct is None:
        return False
    if is_st:
        return pct <= -4.95
    if code.startswith(("300", "301", "688")):
        return pct <= -19.5
    if code.startswith("8") and len(code) == 6:
        return pct <= -29.5
    return pct <= -9.7
```

**容差说明**：
- 0.5%（如 9.7%、19.5%）的容差吸收盘中瞬时价与收盘价的微小差异
- 如果用 `>= 10.0` 这种硬阈值，会漏掉所有 9.7% 以上的"接近涨停"股
- 如果用 `>= 9.5` 太松，会把 +9.6% 这种未涨停股算进来

## 反例：为何不能用统一阈值

```python
# ❌ 错误：所有股票统一 10%
limit_up_count = sum(1 for x in stocks if x['pct'] >= 9.5)
# 2026-06-29 实测问题：
#   - 实际涨停 17 只，其中 11 只是 20% 涨停（创业板/科创板）
#   - 用 9.5% 阈值会漏掉 9.7%~9.99% 的"接近涨停"但实际未涨停股
#   - 用 10.0% 阈值会漏掉所有 9.5%~9.99% 涨停股
# 正确做法：按代码前缀分类判断
```

## 涨跌停统计的"全市场" vs "板块筛选"区别

| 用途 | 数据源 | 备注 |
|------|-------|------|
| 全市场涨停统计 | 东财 push2 API `fs=m:0+t:6,m:0+t:13,m:0+t:80,m:1+t:2,m:1+t:23` | 覆盖沪深京全市场 |
| 板块内涨停股 | 用板块涨跌幅 + 个股涨跌幅交叉验证 | 不要用板块涨幅推算个股涨幅 |

## ST 股识别

腾讯 API 返回的股票名称字段（`parts[1]`）中：
- 名称前带 `ST` 或 `*ST` 前缀的就是 ST 股
- 例：`*ST 仕净`（sz301030）、`ST 荣科`（sz300290）
- ST 股涨停是 ±5%，但停牌 1 天后复牌常出现"首日不设涨跌幅"或"±5%"两种情况，统计时需关注公司公告
