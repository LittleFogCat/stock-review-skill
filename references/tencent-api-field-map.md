# 腾讯行情 API 字段索引完整对照

`http://qt.gtimg.cn/q=<codes>` 是 stock-review-skill 最高频的数据源。**A 股、港股、美股、ETF、指数**的字段索引各不相同，把 A 股解析代码直接套到美股上会导致字段全错。本文件汇总所有常见代码族的字段映射，作为排错时的事实参考。

## 编码说明

- **所有 payload 都是 GBK 编码**，不是 UTF-8：必须 `.decode("gbk", errors="replace")`
- 每行格式：`v_<code>="<payload>"`，payload 用 `~` 分隔
- 字段数因代码族而异（美股 71 段、A 股指数 32 段左右），取数前必须 `if len(parts) > N` 守卫

## A 股指数（`sh000001`、`sz399001` 等）

`v_sh000001="1~上证指数~000001~4027.26~4100.45~4040.12~13210681~..."`

| 索引 | 含义 | 示例 |
|------|------|------|
| `[1]` | 中文名称 | `上证指数` |
| `[2]` | **当前价** | `4027.26` |
| `[3]` | 昨收 | `4100.45` |
| `[4]` | 今开 | `4040.12` |
| `[5]` | 成交量（手） | `13210681` |
| `[6]` | 外盘 | `...` |
| `[7]` | 成交额（**不可靠**） | `0` |
| `[30]` | 时间戳 | `2026-06-26 15:00:00` |
| `[31]` | 涨跌额（绝对值） | `-73.19` |
| `[32]` | 涨跌幅（百分比，已带正负号） | `-1.79` |

**注意**：`[7]` 成交额字段对所有主要指数（A 股 + 港股 + 美股）都返回 `0` 或空，**不要用此字段估算市场总成交额**。

## A 股个股（`sh600519`、`sz002281` 等）

`v_sh600519="1~贵州茅台~600519~1490.50~1480.20~..."`

字段索引与 A 股指数基本一致：
- `[1]` 名称 / `[2]` 现价 / `[3]` 昨收 / `[4]` 今开 / `[5]` 成交量
- `[30]` 时间戳 / `[31]` 涨跌额 / `[32]` 涨跌幅%

**附加字段**（部分个股可用）：
- `[33]` 换手率 / `[34]` 市盈率(动) / `[35]` 市净率 / `[38]` 总市值 / `[39]` 流通市值

## 港股（`hk00700`、`hk09988` 等）

`v_hk00700="1~腾讯控股~00700~..."`

字段顺序与 A 股基本相同，但 `[32]` 涨跌幅单位是 `%`，**数字格式可能为小数**（如 `+0.85` 而非 `+0.85%`），需手动追加 `%`。

## 美股指数（`usDJI`、`usIXIC`、`usINX`）

**与 A 股最大的区别：`[3]/[4]` 不是「昨收/今开」，而是「当前价/昨收」，涨跌幅被推到 `[31]/[32]`。**

| 索引 | 含义 | 示例 |
|------|------|------|
| `[0]` | 状态码 | `200` |
| `[1]` | 中文名称 | `道琼斯` |
| `[2]` | 交易所代码后缀 | `.DJI` |
| `[3]` | **当前价** | `51876.11` |
| `[4]` | **昨收** | `51920.62` |
| `[5]` | 今开 | `51803.77` |
| `[6]` | 成交量 | `1321068148` |
| `[30]` | 时间戳（美东） | `2026-06-26 17:11:58` |
| `[31]` | **涨跌额** | `-44.51` |
| `[32]` | **涨跌幅（百分比，已带正负号）** | `-0.09` |

**典型翻车场景**：把 A 股的 `[2]=现价` 套到美股上 → 取到的是 `.DJI` 字符串；把 `[3]=昨收` 套到美股上 → 取到的是当前价 51876.11 这种大数字，以为涨跌幅是「51876.11%」。

## ETF（`sh510300`、`sz159915` 等）

字段顺序与 A 股个股相同。`[33]` 字段可能是折溢价率。

## 跨族对比速查表

| 代码族 | 现价索引 | 昨收索引 | 涨跌幅索引 | 编码 |
|--------|----------|----------|------------|------|
| A 股指数 | `[2]` | `[3]` | `[32]` | UTF-8 |
| A 股个股 | `[2]` | `[3]` | `[32]` | UTF-8 |
| 港股 | `[2]` | `[3]` | `[32]` | UTF-8 |
| **美股指数** | **`[3]`** | **`[4]`** | **`[32]`** | **GBK** |
| **美股个股** | **`[3]`** | **`[4]`** | **`[32]`** | **GBK** |
| ETF | `[2]` | `[3]` | `[32]` | UTF-8 |

**美股是唯一「现价/昨收索引互换」的代码族**——其他所有代码族都用 A 股的 `[2]/[3]` 习惯。

## 排错流程

1. **解码后字段全部空字符串 / 乱码？** → 检查是否用了 UTF-8（应该是 GBK）
2. **涨跌幅字段是字符串（`.DJI` / `.IXIC`）？** → 你在用 A 股的 `[2]` 取美股字段，索引偏移了
3. **涨跌幅是天文数字（如 51876.11）？** → 你在用 A 股的 `[3]` 取美股字段（美股 `[3]` 是当前价）
4. **字段数 < 33？** → 部分冷门代码可能字段更少，加 `if len(parts) > N` 守卫
5. **拿到荒谬值就立刻打印 `parts[:35]`** 逐项对照——直接看原始 payload 是最快的排错方式

## 参考代码片段

```python
import re, urllib.request

def fetch_tencent_quotes(codes: list[str]) -> dict:
    """通用腾讯 API 拉取，自动按代码族决定字段映射"""
    url = "http://qt.gtimg.cn/q=" + ",".join(codes)
    req = urllib.request.Request(url, headers={"User-Agent": "Mozilla/5.0"})
    with urllib.request.urlopen(req, timeout=20) as resp:
        raw = resp.read().decode("gbk", errors="replace")

    results = {}
    for line in raw.strip().split("\n"):
        m = re.match(r'v_(\w+)="([^"]+)"', line)
        if not m:
            continue
        code, payload = m.group(1), m.group(2)
        parts = payload.split("~")
        if len(parts) < 33:
            continue

        is_us = code.startswith("us")
        # 美股 [3]=现价 [4]=昨收；其他 [2]=现价 [3]=昨收
        cur_idx = 3 if is_us else 2
        prev_idx = 4 if is_us else 3

        try:
            results[code] = {
                "name": parts[1],
                "close": float(parts[cur_idx]) if parts[cur_idx] else 0,
                "prev_close": float(parts[prev_idx]) if parts[prev_idx] else 0,
                "change": float(parts[31]) if parts[31] else 0,
                "changePercent": float(parts[32]) if parts[32] else 0,
                "timestamp": parts[30] if len(parts) > 30 else "",
            }
        except (ValueError, IndexError):
            pass
    return results
```

## 板块代码（`pt01801xxx`）覆盖不全

腾讯行情 API 的板块代码格式 `pt01801xxx`（如 `pt01801110`）覆盖不全：

- **2026-06-08 实测**：33 个板块中仅 13 个返回数据
- **2026-06-25 实测**：`pt01801110` 实际返回「家用电器」板块——代码与名称映射与历史文档不一致

**结论**：pt 板块代码**仅作辅助**（板块名称参考），**不作为板块涨跌幅数据源**。板块涨跌幅一律走 Sina `/tob/` 文章 + 腾讯 API 个股推算 + 东财 push2 API（如可用）。

## 早盘周末美股含义陷阱

### 场景

周一（08:00）触发早盘快报，前一交易日是周五（中间周六周日 A 股+美股均休市）。

### 误判

以为「隔夜美股」指周日晚上→周一凌晨的美股。但**美股周六周日休市**，周日没有任何美股交易。「隔夜美股」严格意义上等于**上周五收盘数据**，且这是周一开盘前唯一的可参照美股基准。

### 实测（2026-06-29）

腾讯 API 返回美股时间戳 `2026-06-26 17:11:58`（上周五美东收盘），道指 -0.09%、纳指 -0.24%、标普 -0.05%。

### 早盘触发日 vs 美股数据时间窗口

| 早盘触发日 | A 股前一交易日 | 美股「隔夜」= | 实际数据时间戳 |
|----------|--------------|-------------|--------------|
| 周一（08:00） | 上周五 | 上周五收盘 | 上周五 17:00 美东 |
| 周二（08:00） | 周一 | 周一收盘 | 周一 17:00 美东 |
| 周三（08:00） | 周二 | 周二收盘 | 周二 17:00 美东 |
| 周四（08:00） | 周三 | 周三收盘 | 周三 17:00 美东 |
| 周五（08:00） | 周四 | 周四收盘 | 周四 17:00 美东 |

### 正确处理

1. 早盘快报中明确标注美股数据为「上周五收盘（因周末美股休市）」或类似措辞
2. 不要尝试拉取「真正隔夜」美股——没有这个数据
3. 复盘日期标签与美股数据日期对应：报告写「2026-06-29 早盘快报」，美股数据写「6/26 周五收盘」
4. 关注板块/个股可基于「周五收盘 + 周末消息」综合判断

### 易错点

周一早盘写「隔夜美股 6/26 数据」可能被读者误解为「6/26 收盘当天的盘前」——需在报告中说明「6/26 周五收盘=周末前最新美股基准」。
