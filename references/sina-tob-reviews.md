# Sina 综合收评文章（/tob/）— 当日复盘首选数据源

Sina 财经在 `/tob/` 路径下发布的**综合收评文章**，单篇文章即可覆盖当日复盘所需的指数、板块、个股、消息面等核心数据。该文档详细说明文章结构、数据提取方法、与东财 push2 API 的分工。

---

## URL 模式与定位

```
https://finance.sina.com.cn/tob/YYYY-MM-DD/doc-*.shtml
```

示例（2026-06-25）：
```
https://finance.sina.com.cn/tob/2026-06-25/doc-inierayt7775708.shtml
```

**文章性质**：由 Sina 编辑团队于收盘后 15:00-16:00 发布的盘后综合评述，标题通常以「收评：」开头。

**与同类文章的关系**：

| 文章路径 | 视角 | 内容特点 |
|---------|------|---------|
| `/stock/bxjj/` | ETF 基金维度 | 各板块 ETF 精确涨跌幅（如「证券ETF华富+9.53%」），适合 ETF 投资者视角 |
| `/stock/snipe/` | 盘中分析 | 板块/概念实时分析，盘中即可发布 |
| **`/tob/`** | **大盘总览** | **三大指数 + 涨跌家数 + 板块排名 + 涨停龙头 + 消息面分类，最接近标准复盘结构** |

## 文章内容结构（实测样本）

2026-06-25 收评文章（标题「收评：创业板指涨近3% 存储芯片板块全线走强」）实际返回 2562 字符的正文，包含以下结构：

### 1. 开篇 — 核心盘面（1-2 段）
```
6月25日消息，市场震荡走强，创业板指涨近3%，科创50大涨近4%。
板块方面，存储芯片板块全线走强，兆易创新、德明利、佰维存储、香农芯创、
普冉股份、江波龙六家千亿市值企业股价同步创下历史新高；PCB概念反复活跃，
中材科技、红板科技、超声电子涨停；大金融板块全天活跃，长江证券涨停；
超级电容概念走强，黑猫股份5天4板。下跌方面，贵金属板块持续调整，
兴业银锡跌超6%；油气股下挫，新潮能源领跌。总体来看，两市个股呈普跌态势，
下跌个股超4200只。
```

**提取价值**：
- 三大指数涨跌幅
- 涨跌家数（「下跌个股超4200只」→ 上涨约 1100 只）
- 热点板块 TOP3-4（含具体龙头股）
- 领跌板块 TOP2-3（含具体跌幅领跌股）
- 市场情绪定性（「指数牛、个股熊」类描述）

### 2. 收盘数据
```
截至收盘，沪指报4120.28点，涨0.23%，深成指报16344.08点，涨1.82%，
创指报4371.99点，涨2.84%。
```

**提取价值**：精确的三大指数收盘价（与腾讯 API 交叉验证）

### 3. 板块涨幅/跌幅概述
```
盘面上，机场航运、元件、证券板块涨幅居前；贵金属、芬太尼、教育板块跌幅居前。
```

**提取价值**：排序好的板块列表，可直接用于 `topSectors[].name` 和 `fallingSectors[].name`

### 4. 热点板块详细描述（含消息面归因）
```
热点板块：
1、存储芯片
兆易创新、德明利、佰维存储、香农芯创、普冉股份、江波龙六家千亿市值企业
股价同步创下历史新高。
美东时间周三盘后，存储巨头美光科技公布的第三财季财报显示，公司继续受益
于人工智能热潮带来的旺盛需求，第三财季营收增长高达346%，毛利率更是高达
84.9%，远超市场预期。
美光CEO桑杰·梅赫罗特拉在财报电话会上表示，当前DRAM与NAND闪存行业需求
持续大幅高于行业供给...
```

**提取价值**：
- 热点板块 TOP 详细描述（2-3 段）
- 直接的消息面归因（美光财报 → A股存储芯片爆发，因果链完整）
- 机构观点引用

### 5. 消息面分类
```
消息面：
【建设银行：将于7月24日日终清算时起关闭代理上海黄金交易所个人贵金属交易业务功能】
【联合国报告称全球仍有6.55亿人用不上电】
【金饰克价年内跌近500元】
【广东：按规定将脑机接口相关服务纳入医疗服务价格项目...】
```

**提取价值**：
- 单独的 `消息面` 章节已按条目列出
- 适合填充 `news[].content[]`（可按公司/政策/行业分组）

### 6. 机构观点
```
机构观点：
中国银河证券策略首席分析师杨超认为，下半年投资环境或有所改善...
中泰证券研报称，未来科技或依然是市场主线...
```

**提取价值**：机构观点摘录，可放入「消息面」或「机构观点」章节

## 数据提取方法

### Step 1：从 Sina 首页找到文章 URL

```python
import urllib.request, re

url = "https://finance.sina.com.cn/stock/"
req = urllib.request.Request(url, headers={"User-Agent": "Mozilla/5.0"})
with urllib.request.urlopen(req, timeout=15) as resp:
    html = resp.read().decode('utf-8', errors='replace')

# 查找所有路径的收评文章
for pattern, label in [
    (r'href="(https://finance\.sina\.com\.cn/stock/bxjj/[^"]+)"', 'bxjj'),
    (r'href="(https://finance\.sina\.com\.cn/stock/snipe/[^"]+)"', 'snipe'),
    (r'href="(https://finance\.sina\.com\.cn/tob/[^"]+)"', 'tob'),
]:
    for href in re.findall(pattern, html):
        if 'YYYY-MM-DD' in href:  # 替换为目标日期
            print(f"{label}: {href}")
```

**注意**：
- 用 `YYYY-MM-DD` 日期字符串过滤，避免拿到历史文章
- 同一天可能有多篇 /tob/ 文章（如 8:00 早间 + 16:00 收盘），收盘后应取最后发表的那篇
- 标题含「收评」字样的是综合收评（综合大盘），标题含「ETF收评」的是 bxjj 路径下的 ETF 视角

### Step 2：提取文章正文

```python
import urllib.request, re, html as html_lib

url = "https://finance.sina.com.cn/tob/2026-06-25/doc-inierayt7775708.shtml"
req = urllib.request.Request(url, headers={"User-Agent": "Mozilla/5.0"})
with urllib.request.urlopen(req, timeout=15) as resp:
    raw_html = resp.read().decode('utf-8', errors='replace')

# Sina 文章正文通常在 <p> 标签中
# 也可能包裹在 div id="artibody" 中
paragraphs = re.findall(r'<p[^>]*>(.*?)</p>', raw_html, re.DOTALL)
body_text = '\n'.join(paragraphs)
# Strip HTML
body_text = re.sub(r'<[^>]+>', '', body_text)
body_text = html_lib.unescape(body_text)
body_text = re.sub(r'\s+', ' ', body_text).strip()
```

**为什么用 `<p>` 标签 fallback**：Sina 不同栏目文章的结构不一致，`div id="artibody"` 不是 100% 存在。`<p>` 标签 fallback 在所有栏目的成功率最高。

### Step 3：从正文填充复盘字段

```python
# 提取热点板块名称（"X、Y、Z板块涨幅居前"）
sector_match = re.search(r'盘面上，(.*?)板块涨幅居前', body_text)
if sector_match:
    top_sectors_names = [s.strip() + '板块' for s in sector_match.group(1).split('、')]

# 提取跌幅板块名称
falling_match = re.search(r'跌幅居前', body_text)
# 跌幅板块通常需要更宽松的匹配

# 提取涨跌家数
count_match = re.search(r'下跌个股超?(\d+)只', body_text)
if count_match:
    down_count = int(count_match.group(1))
```

## 适用场景

### 适合用 /tob/ 文章的情况
- ✅ **当日复盘**核心数据采集（首选）
- ✅ 需要快速获取热点板块 + 龙头股清单 + 消息面
- ✅ 需要消息面归因（美光财报 → 存储芯片爆发 这类因果链）
- ✅ 浏览器不可用 / 冻结时的回退方案
- ✅ 文章中已含部分「政策面」「产业面」「海外市场」分类消息

### 不适合用 /tob/ 文章的情况
- ❌ 早盘快报（需要的是昨日复盘 + 隔夜消息，不是今日盘中收评）
- ❌ 需要精确板块涨跌幅数字（文章只说「板块涨幅居前」不说具体数字——这部分需东财 push2 API 或腾讯 API 个股）
- ❌ 文章本身缺失时（小盘日期/节假日可能无 /tob/ 文章）
- ❌ 需要获取每只个股的精确涨跌幅（文章只列出龙头股名称）

## 与其他 Sina 文章的互补关系

| 场景 | 首选文章 | 辅助文章 |
|------|---------|---------|
| **当日复盘（盘后）** | `/tob/` 综合收评 | `/cpbd/` 操盘必读（消息面更全） |
| **早盘快报** | `/cpbd/` 操盘必读 + `/y/` 财经早报 | `/bxjj/` ETF收评（隔夜美股） |
| **盘中分析** | `/snipe/` 实时分析 | `/tob/`（盘后才有） |
| **板块 ETF 数据** | `/bxjj/` ETF收评 | `/tob/`（仅部分含 ETF 视角） |
| **公司公告** | 东财公告 API（首选） | `/cpbd/` 含部分公司公告 |

**典型 cron 复盘流程**（2026-06-25 验证）：
1. 抓取 `/stock/` 首页 → 找到 `/tob/YYYY-MM-DD/doc-*.shtml` 链接
2. 抓取该文章 → 提取正文（~2500-3500 字符）
3. 从正文提取：三大指数收盘价（交叉验证腾讯 API）、热点板块名称、龙头股清单、消息面条目
4. 抓取腾讯 API 个股（~80 只，覆盖龙头股）→ 获得精确涨跌幅
5. 抓取东财 push2 API（可选）→ 获得全部板块精确涨跌幅（用于填充 `changePercent`）
6. 组装 JSON + markdown

**时间消耗**：~30 秒（4 次 HTTP GET + 1 次本地 JSON 序列化）

## 已知风险与应对

### 风险 1：文章内容结构不一致

不同栏目（tob / bxjj / snipe / cpbd / y）的文章 HTML 结构不一致。`<p>` 标签 fallback 通用但可能混入导航、广告等无关段落。

**应对**：在解析后用关键词过滤（如只保留含「板块」「涨」「跌」「公司」等关键词的段落），或用更严格的选择器。

### 风险 2：文章标题诱导性内容

部分文章标题含「ETF收评」「收评」「早报」等关键词，但实际内容可能是专题报道或单只股票分析。

**应对**：
- 标题必须包含「收评」字样（且不含「ETF收评」单只视角）
- 标题长度通常 30-60 字符
- URL 日期路径必须匹配目标日期

### 风险 3：同一天多篇 /tob/ 文章

交易日可能出现早间收评 + 收盘收评两篇 /tob/ 文章。

**应对**：取最后一篇（按 doc ID 后缀字母排序最大者），或匹配「收盘后发布」时间窗口（如 15:00 之后）。

### 风险 4：节假日无 /tob/ 文章

A 股休市日（周末 + 法定节假日）不会有 /tob/ 文章发布。

**应对**：节假日复盘回退到上一交易日数据，提示用户「今日因XX休市，复盘最近交易日」。
