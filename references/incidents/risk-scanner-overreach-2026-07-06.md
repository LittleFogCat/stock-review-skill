# risk_signal_scanner.py 过度脚本化回滚（2026-07-06）—— 不是所有规则都该用代码写死

> **事故级别**：🟡 方法论判断失误（无实际数据损失，但浪费开发时间）
> **事故性质**：**反向**的事故——错误的不是「该脚本化的没脚本化」，而是「不该脚本化的硬塞了脚本」

## 事件回顾

2026-07-06 SKILL.md 精简 review 时，把「风险信号触发机制」识别为 **P1 脚本化机会**，新建 `scripts/risk_signal_scanner.py`（约 200 行）。脚本接受 `signals.json`，机械匹配：

- 数字信号：美股/商品/汇率/VIX 阈值判断
- 语义信号：地缘冲突 / 央行异动 / 政策博弈关键词匹配
- 输出 R1-R4 风险等级 + 触发条目清单

主人逐步追问后识别出**本质问题**：风险评估是**语义整合任务**，不是规则匹配任务。脚本随即被删除。

## 这个判断错在哪

| 维度 | 我的假设 | 实际情况 |
|------|---------|---------|
| 任务性质 | 「30 条触发清单 = 可以脚本化」 | 30 条触发清单是**人类经验总结**，给 LLM 阅读用，不是给机器执行的 |
| AI 能力 | 「LLM 自己总结会漏」 | LLM 会漏，但解药是「强化 LLM 自检意识」而非「写脚本替代 LLM」 |
| 输入来源 | 「signals.json 是 validated schema」 | inputs 从哪来？——**这个问题没答案**，下游无人接 |
| 关键词维护 | 「关键词覆盖足够」 | 日本 9 级地震、福岛核电站堆芯熔毁——自然灾害类关键词完全缺失 |
| 输出价值 | 「脚本输出可追溯、可审计」 | 脚本输出后**无人消费**——落盘、上报、嵌入 markdown 都没接 |

## 跟项目里其他决策点的对比

判断「什么该脚本化、什么该让 AI 思考」的原则：

| 任务类型 | 适合脚本化？ | 例子 |
|---------|------------|------|
| **数字阈值判断**（机械、确定）| ✅ | 美股 -3% → R1、changePercent 必须是 number |
| **结构化校验**（黑名单、字段名）| ✅ | markets 必须是对象、focusSectors 字段名是 name |
| **事实查询**（K 线、日历、API 响应）| ✅ | 用腾讯 K 线判断交易日、用日历算法算跨周末美股日期 |
| **模糊措辞拦截**（黑名单字面匹配）| ✅ | source 不许「估算」、文本不许「估值偏高」 |
| **语义整合**（多源信号聚合+语境理解）| ❌ | 风险等级评估、地缘政治影响、产业政策连锁反应 |
| **叙事生成**（措辞、解释、推荐）| ❌ | 早盘关注板块的「短线关注理由」、新闻摘要的重组 |
| **未知信号识别**（新型事件）| ❌ | 大地震、疫情、突破性政策——硬编码跟不上现实 |

## 教训

1. **过度脚本化的典型征兆**：「**这一步看起来机械，但脚本写不出来具体怎么调**」
   - 风险扫描器写出 200 行时，我应该警觉：signals.json 谁填？填错了谁兜底？
   - 这两个问题答不上来 = 脚本没找对边界
2. **「自动化偏好」要克制**：开发一个工具比写一段文字说明更有成就感（多巴胺反馈），但工具的边际价值可能远低于文字
3. **铜关税事件的诊断错位**：当时归因「LLM 没扫够信号」，但根因是「**风险评估被简化成触发清单扫描**」——写清单给 LLM 读是对的，但**用代码复制这份清单是错的**
4. **「30 条触发条件」是人脑索引**：LLM 读 1.9 章节能 trigger 内置语义识别，脚本读关键词只能 trigger 字面匹配——**这是质的差距，不是量的差距**

## 当时主要发现的脚本盲点（也佐证了判断）

跑测试时主人指出：
- 「**日本发生 9 级大地震，会怎么样？**」——脚本关键词只有「军事冲突/战争/制裁生效/核试验/战时状态」，**地震/海啸/核事故完全空白**
- 这个边界 case 一旦真实发生，脚本 R1 不触发，LLM 看脚本输出会以为「今天没风险」，反而掩盖了真实风险

## 正确的处理方式

- **风险评估环节维持 SKILL.md 1.9 文字机制**，由 LLM 自觉扫描
- **如需辅助**，可在 1.9 末尾加一条：「数字信号快速扫（美股、商品、汇率阈值）、语义信号仔细看（地缘、政策、央行意图）、不确定时升级一档」——这是 **AI 思考指南**而非 **脚本输出格式**
- **`scripts/risk_signal_scanner.py` 已删除**，未来不重新实现
